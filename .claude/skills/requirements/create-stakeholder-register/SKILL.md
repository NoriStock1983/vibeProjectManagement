# ステークホルダー登録簿作成スキル（PMBOK第8版準拠）

ステークホルダー登録簿（Stakeholder Register）を作成し、Markdownファイルとして保存する。

## 目的

「誰の要件を収集すべきか」を明確にするための文書。PMBOK第8版ではステークホルダーとのエンゲージメントが原則として重要視されており、要件漏れを防ぐために必須となる。

## 入力情報の収集

以下の情報が揃っていない場合は、ユーザーにまとめて確認する（一度の質問で済ませる）：

各ステークホルダーについて：
1. 氏名・役職・所属組織
2. プロジェクトへの関与度（高・中・低）
3. 影響力（プロジェクトの意思決定への影響）
4. 関心事（何に興味・懸念を持っているか）
5. 要件収集における役割（インタビュー対象者・承認者・情報提供者など）
6. コミュニケーション方法（会議・メール・報告書など）
7. コミュニケーション頻度

プロジェクト憲章にステークホルダー情報が含まれている場合は参照する。

## 出力形式

ファイル名：`stakeholder-register_[プロジェクト名].md`
保存先：`vibeProjectManagement/docs/requirements/`

テンプレートは [assets/template.md](assets/template.md) に定義されている。必ずこのファイルを読み込み、その構造に従って出力する。

## 作成の指針

- **網羅的に洗い出す** — 直接関与するステークホルダーだけでなく、影響を受ける関係者（エンドユーザー・法務・情報セキュリティ・運用チームなど）も含める
- **関与度×影響力マトリクスを活用** — 各ステークホルダーの優先度を視覚化する
- **コミュニケーション計画の基礎にする** — 各ステークホルダーへのアプローチ方法を明記する
- PMBOK第8版の「ステークホルダー・パフォーマンスドメイン」に沿って、エンゲージメントのレベル（抵抗・中立・支持・リード）を評価する

## 作成後の処理

ファイル保存後、ユーザーに保存パスを伝え、「内容の確認・修正があればお知らせください」と案内する。
