# WBS作成スキル（PMBOK第8版準拠）

WBS（Work Breakdown Structure / 作業分解構造図）を作成し、Markdownファイルとして保存する。

## 目的

要件を実際の作業・成果物へと落とし込む橋渡しとなる文書。PMBOK第8版のスコープ・パフォーマンスドメインにおいて、成果物の定義と紐付けて管理される。WBSは「何を作るか（成果物）」を中心に分解し、「作業パッケージ」レベルまで展開する。

## 入力情報の収集

以下の情報が揃っていない場合は、ユーザーにまとめて確認する（一度の質問で済ませる）：

1. 分解の対象（プロジェクト全体 / 特定フェーズ）
2. プロジェクトスコープ記述書の成果物一覧（あれば参照）
3. 開発手法（ウォーターフォール / アジャイル / ハイブリッド）
4. WBSの階層の深さ（第何レベルまで展開するか）
5. 各作業パッケージの担当組織・担当者（わかる範囲で）

## 出力形式

ファイル名：`wbs_[プロジェクト名].md`
保存先：`vibeProjectManagement/docs/requirements/`

テンプレートは [assets/template.md](assets/template.md) に定義されている。必ずこのファイルを読み込み、その構造に従って出力する。

## WBSの設計原則

- **100%ルール** — 上位レベルの成果物は、すべての子要素（作業パッケージ）の合計で100%構成されること。何かが漏れていないか、何かが重複していないかを確認する
- **成果物指向** — 「〜する」（アクション）ではなく「〜を作る」（成果物）を中心に分解する
- **作業パッケージの粒度** — 最下層の「作業パッケージ」は、1人の担当者が数日〜2週間程度で完了できる単位を目安とする
- **WBS番号の付与** — 1.0, 1.1, 1.1.1 のように階層構造がわかる番号を付与する

## 作成後の処理

ファイル保存後、ユーザーに保存パスを伝え、「内容の確認・修正があればお知らせください」と案内する。
要件定義Agent全体のフローを実行中の場合は、完了レポートを出力する。
