# VibePM - Claude 作業ガイド

Claude を活用して、仕様書・議事録・WBS・週次報告などの定型的なプロジェクト管理作業を担うプロジェクト。

## 作業前に必ず読むこと

以下のファイルを読み、プロジェクトの文脈を把握してから作業を開始すること：

| ファイル | 内容 |
| --- | --- |
| `docs/project/overview.md` | プロジェクト概要・目的・フェーズ |
| `docs/project/stakeholders.md` | ステークスホルダー一覧・役割 |
| `docs/project/schedule.md` | スケジュール・マイルストーン |
| `docs/project/architecture.md` | アーキテクチャ情報 |

## 作業指示

### 自律的に実行してよいこと

- 指定されたテンプレート・スキルに従ったドキュメントの新規作成
- コーディング・テストコードの作成・修正
- 既存ドキュメントの誤字・表記揺れの修正

### 実行前に確認すること

- 既存ドキュメントの構成・方針の変更
- ファイルの削除・移動
- `docs/project/` 配下のファイル更新（プロジェクト情報の変更）

## 成果物の保存先

| 成果物 | 保存先 |
| --- | --- |
| 議事録 | `docs/minutes/` |
| 週次報告 | `docs/weekly-reports/` |
| 仕様書・要件定義 | `docs/specs/` |
| WBS | `docs/wbs/` |
| ADR | `docs/adr/` |
