# ADR-0002: .NET 10 採用

## ステータス

採用

## 背景

新規C#プロジェクトのターゲットフレームワークを決定する必要があった。当初は最新版である.NET 11の採用を検討したが、リリーススケジュールおよびツールチェーンの対応状況を踏まえて再検討した。

開発環境にはVSCodeを採用し、カバレッジ計測にCoverlet + ReportGenerator、ミューテーションテストにStryker.NETを使用する方針（ADR-0001）のため、各ツールの安定サポート状況もフレームワーク選定の判断材料とした。

## 検討した選択肢

### 選択肢 A: .NET 11

- **概要**：2026年11月リリース予定のメジャーバージョン
- **メリット**：最新機能・パフォーマンス改善を最速で取り込める
- **デメリット**：2026年4月時点ではプレビュー版のため正式サポートなし。Coverlet・Stryker.NETなどのツールチェーンが未対応の可能性が高く、プロジェクト初期段階での採用はリスクが大きい

### 選択肢 B: .NET 10（LTS）

- **概要**：2025年11月に正式リリースされたLTS（Long Term Support）バージョン
- **メリット**：正式リリース済みで安定稼働が確認されている。LTSのため2028年5月まで長期サポートが保証されている。Coverlet・Stryker.NET・C# Dev Kitなど主要ツールチェーンが安定サポート済み
- **デメリット**：.NET 11の新機能は利用できない

### 選択肢 C: .NET 9

- **概要**：2024年11月リリースの標準サポート版
- **メリット**：実績・安定性が高い
- **デメリット**：標準サポート版（非LTS）のためサポート期間が短い（2026年5月終了予定）。新規プロジェクトとしてサポート期間が不十分

## 決定

**.NET 10 を採用する。**

正式リリース済みのLTSバージョンであり、Coverlet・Stryker.NET・C# Dev Kitなど採用予定のツールチェーンが全て安定サポートしている。2028年5月まで長期サポートが保証されており、新規プロジェクトの基盤として適切な選択である。

.NET 11については正式リリース後（2026年11月予定）にツールチェーンの対応状況を確認した上で、アップグレードを再検討する。

## 影響

### ポジティブな影響

- ツールチェーン（Coverlet・Stryker.NET・C# Dev Kit）の安定サポートが保証される
- LTSによる長期サポート（〜2028年5月）でプロジェクト継続性が確保できる
- 正式リリース済みのため、ドキュメント・コミュニティリソースが充実している

### ネガティブな影響・トレードオフ

- .NET 11の新機能・パフォーマンス改善は即時利用できない

### 対応が必要なタスク

- [ ] プロジェクトファイル（.csproj）の `TargetFramework` を `net10.0` に設定
- [ ] .NET 11 正式リリース後のアップグレード検討をスケジュールに組み込む
